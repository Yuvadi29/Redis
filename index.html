Deploy to GitHub Pages

1 Compile the Site

Make sure you have compiled the latest Markdown docs into `index.html` by running the build script in your terminal:

node build.js

2 Initialize Git Repository

If you haven't already, initialize a git repository in this project folder, stage all your files (including `index.html`, `build.js`, and all the subdirectories with their images), and commit them:

git init
git add .
git commit -m "feat: Add Claude-themed Redis documentation hub"

3 Create GitHub Repository & Push

Create a new repository on GitHub (keep it public). Link it to your local repository and push your changes to the main branch:

git remote add origin https://github.com/your-username/redis-learning-hub.git
git branch -M main
git push -u origin main

4 Enable GitHub Pages

Go to your repository on GitHub.com and perform the following configuration:

1. Click on the Settings tab at the top right of the repo.

2. In the left sidebar, click on Pages (under the Code and automation section).

3. Under "Build and deployment", set the Source to Deploy from a branch.

4. Set the branch to main and the directory to / (root).

5. Click the Save button.

5 Profit!

GitHub will automatically build and publish your site. In 1-2 minutes, your website will be live at:

https://your-username.github.io/redis-learning-hub/
